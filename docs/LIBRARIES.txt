# TickerTouch — Required Libraries
# Install all via Arduino Library Manager unless noted

[REQUIRED]
bb_captouch          by Larry Bank    — capacitive touch (GT911)
                                        Search "bb_captouch" in Library Manager

GFX Library          >= 1.6.1         — "GFX Library for Arduino" by moononournation
                                        Library Manager

LVGL                 8.3.x            — search "lvgl" in Library Manager
                                        USE 8.3.x ONLY — do NOT install 9.x
                                        Breaking API changes in 9.x will cause build errors

ArduinoJson          6.21.x           — Library Manager
                                        USE 6.x ONLY — do NOT update to 7.x

ESP32 Arduino Core   >= 3.x           — Boards Manager
                                        URL: https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json

[lv_conf.h]
Copy lv_conf.h from this project to: Arduino/libraries/lv_conf.h
Place it one level ABOVE the lvgl/ folder, NOT inside it.
This file is pre-configured with:
  - LV_USE_CANVAS 1       (required)
  - LV_USE_QRCODE 1       (required for captive portal QR)
  - Fonts 12-48pt enabled
  - LVGL heap routed to PSRAM

[Board Settings — ESP32S3 Dev Module]
Flash Size:        4MB (32Mb)
PSRAM:             OPI PSRAM
Partition:         Huge APP (3MB No OTA / 1MB SPIFFS)
CPU Frequency:     240MHz
USB CDC On Boot:   Enabled    <-- required for Serial Monitor

[Touch Pins — GT911 on JC4827W543]
SDA = 8
SCL = 4
INT = 3
RST = -1  (not used)

[CRITICAL — sdkconfig.ini]
CONFIG_SPIRAM_USE_MALLOC is NOT set in sdkconfig.ini.
Setting it causes String/malloc allocations to land in PSRAM
which triggers memcpy crashes during HTTP parsing. Leave it out.

[CRITICAL — HTTP reading]
Never use http.getString() — returns a PSRAM-backed String.
Use httpGET_buf() with useHTTP10(true) and stream->read() instead.
See data_manager.cpp for implementation.
